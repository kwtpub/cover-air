# Архитектура проекта

Проект использует **FCD (Feature-Centric Design)** архитектуру - организация кода по функциональным фичам.

## Структура проекта

```
src/
├── features/               # Основные фичи/секции сайта
│   ├── hero/              # Главный экран
│   ├── problem/           # Секция проблемы
│   ├── solution/          # Секция решения
│   ├── exclusive/         # Эксклюзивные предложения
│   ├── how-it-works/      # Как это работает
│   ├── features-section/  # Возможности
│   ├── comparison/        # Сравнение
│   ├── partners/          # Партнёры
│   ├── faq/              # Часто задаваемые вопросы
│   └── connect-form/      # Форма подключения
│
├── shared/                # Переиспользуемые компоненты
│   └── components/
│       ├── header/        # Шапка сайта
│       └── footer/        # Подвал сайта
│
├── App.tsx               # Главный компонент приложения
├── App.css               # Глобальные стили приложения
├── index.css             # Базовые стили и переменные
└── main.tsx              # Точка входа

```

## Принципы организации

### Features (Фичи)
Каждая фича представляет собой отдельную секцию сайта со своими:
- Компонентом (`.tsx`)
- Стилями (`.css`)
- Точкой экспорта (`index.ts`)

### Shared (Общие компоненты)
Компоненты, которые используются на всех страницах:
- Header - навигация
- Footer - подвал с контактами

## Преимущества FCD

1. **Изолированность** - каждая фича независима
2. **Масштабируемость** - легко добавлять новые фичи
3. **Читаемость** - структура понятна на первый взгляд
4. **Переиспользование** - общие компоненты в shared
5. **Поддержка** - легко найти нужный код

## Импорты

Импорт компонентов через index.ts:

```typescript
// Фичи
import { Hero } from './features/hero';
import { FAQ } from './features/faq';

// Общие компоненты
import { Header } from './shared/components/header';
import { Footer } from './shared/components/footer';
```
